LunarSystem vocabulary

Namespace: https://jeromev.github.io/LunarSystem/ontology#  ·  prefix luna:  ·  Turtle  ·  code

Most LunarSystem content is published with standard vocabularies — schema.org, Dublin Core, FOAF, SIOC, PROV-O. This namespace covers only the CMS-internal system predicates that have no standard equivalent. Draft — terms may change.

Core terms (in the triplestore)

TermLabelMeaning
luna:isActiveis activeWhether a node (page, level, group, user, text, mod) is active/published (1 or 0).
luna:levelaccess levelLinks a node to the access level that gates it.
luna:contentcontentThe raw (HTML) body of a text block, as authored.

Additional system predicates (serialized model)

Emitted in ?output=xml/n3/json; the public-facing equivalents are usually schema.org.

TermLabelMeaning
luna:nidnode idNumeric primary identifier. Also schema:identifier.
luna:lidliteral idSlug-like identifier (user email, page path segment). Also schema:name.
luna:aliasaliasResolved URL path of a page.

The serializer also emits luna:page, luna:text, luna:save_time, and the legacy snake-case luna:is_active (the triplestore form is luna:isActive); these are internal and may be normalised later.

Resource (instance) IRIs are a separate concern — they follow the deployment host (luna::$site_uri, e.g. http://localhost:8080/id/…), not this vocabulary namespace.
